Resumen de Tareas

En esta sesión se aprendió a trabajar con DASH que es la división de los flujos de audio/vídeo en pequeños segmentos que se transportan mediante HTTP. Entre las características está el streaming adaptativo. Para desarrollar esta función, se define un manifiesto es un fichero con extensión. mpd
Vimos dos ejemplos de manifiesto MPEG-DASH.
• ¿Cuántas codificaciones de audio distintas puede utilizar el cliente para adaptarse a condiciones cambiantes de la red?

Manifiesto 2: Tres codificaciones de video distintas para adaptarse a condiciones cambiantes de la red.
• ¿Cuántas resoluciones de vídeo distintas puede usar el cliente? Indica cuáles.
Manifiesto 1: En este caso utiliza la misma resolución de video (854x480).
Manifiesto 2: En este caso utiliza tres resoluciones de video (512x288), (768x432), (1280x720).
Para continuar con el desarrollo de la clase práctica vamos a montar un servicio de vídeo bajo demanda utilizando MPEG-DASH.
Para ellos utilizaremos una máquina virtual GNU/Linux para servir los contenidos multimedia. La máquina virtual creada tiene configurada la red como "NAT" para ellos incorpora una regla para redirigir el tráfico TCP dirigido hacia el puerto 60022 hacia el puerto 22.
Posteriormente para realizar manifiesto necesitamos Instalar el paquete GPAC en nuestra máquina virtual con el siguiente par de comandos:sudo apt-get update y sudo apt-get install gpac. Para utilizar un fichero de vídeo es necesario demultiplexar audio y vídeo y lo logramos con el paquete ffmpeg ya con los ficheros en el estado que deseamos, podemos con la herramienta MP4Box generar el manifiesto.
